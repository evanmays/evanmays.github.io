Crypto Native Games

You don't need crypto to play crypto-native games. That's the first thing i'll tell a layperson because it flips their understanding on its head.

I don't like ponzies, but I think sustainable crypto-native games are cool. Emphasis on less gambling and more on new game mechanics for new ways to have fun.

Making Fortnite skins into NFTs is not new game mechanics

New game mechanics come from different properties of the blockchain. It's not always about decentralization, but for some game mechanics it could be. Other properties to not forget are interopability, permissionlessness, credible commitments, and data availability.

Here's a reading list that I feel aligned with

  • Gubsheep: The Strongest Crypto Gaming Thesis has a good (and the original?) definition of "crypto-native games"
  • Guilty Gyoza at ETHSeoul: What is Onchain Reality?
  • Into the Bytecode Justin Glibert: Ember, and building onchain games.
  • WLD Show Podcast: Dark Forest v0.6 Round 5: The Junk Wars Saga. A Full Oral History as Told by dfdao
Here's the mechanics I've seen so far which wouldn't be possible w/o a blockchain. (or require too much game developer effort)
  • User Generated Game Modes (A form of user generated content): Some games let users make custom game modes with customer rules (i.e. Halo), but once players can see the entire source code of your backend, they can make fundamental changes to the game. This happened in Dark Forest with DFDao creating Dark Forest Arenas. The normal Dark Forest game is a weeks long strategy game. Arenas is sped up in every way possible such that a match lasts hours.
